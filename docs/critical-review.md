# TetsuLog ガチ鉄目線レビュー

> 「これじゃ駅メモと変わらない」と言われないために。実際の鉄道趣味者が日常で使う粒度・語彙・体験を反映できているか、自分たちで先に粗探しする。

---

## 1. 設計上の弱点（モデル・データ）

### 1-1. 編成番号の表記が雑
- 現状: `Formation.code` は "トウ47" など。だがガチ鉄は **所属区所カナ** に厳密。「トウ」=東京総合車両センター、「ミツ」=三鷹、「マト」=松戸、「クラ」=鎌倉、「カワ」=川越、「サイ」=さいたま、「ヤマ」=山手など。
- JR西は「アカ」(網干)、JR東海は「カキ」(神領)、「シン」(静岡)等。
- 課題: シードJSONは prefix が "T" や "ミツ" 混在。所属区所として `VehicleClass.depots` or `Formation.depot` を明示活用すべき。
- 影響: 「自分が撮ったのはトウなのかカワなのか」が記録できないと、ガチ層からは即離脱対象。

### 1-2. 「車番」レベルの粒度が欠落
- 編成 = 複数の車両（モハ・サハ・クハ等）の集合。ガチ鉄は **車番(クハE235-1247) 単位** で記録する人がいる。
- 現状: `Formation.carNumbers: [String]` はモデルにあるが UI で活用されていない。`Sighting` には車番が紐付かない。
- 課題: 「3号車のサハE235-4624のドア横の傷」みたいなマニアの粒度に届かない。

### 1-3. ヘッドマーク・装飾・特別塗装の記録不能
- 「○周年HM付き」「ラッピング車両」「銀河鉄道塗装」「リバイバルカラー」など、**遭遇時の装飾**を記録したいニーズは強烈。
- 現状: `Sighting` にこのフィールドがない。
- 致命度: 高（情緒価値の中心）。

### 1-4. 列車番号・定期/臨時の区別不能
- ガチ鉄は「2024M」「9501M（臨時）」「回送（試8520）」「配給（配9999）」を記録する。
- 現状: なし。`Sighting` に `trainNumber` `kind`（定期/臨時/回送/試運転/配給/団体）がない。

### 1-5. 撮影時の天候・条件が欠落
- 「曇り」「雨」「夕焼け」「初雪」など、写真の文脈情報。
- 現状: なし。EXIF から取れる撮影設定（SS・F値・ISO）も活用されていない。

---

## 2. UX 上の摩擦（鉄道ファンが必ず怒るやつ）

### 2-1. 検索・フィルタが弱い
- 形式43件で既に「スクロール疲労」が出始める。本番で200形式になったら破綻。
- 必要: 形式名インクリメンタル検索、事業者フィルタ（JR東/西/海/九/貨物/私鉄）、「廃車進行中」「未遭遇」「ラストラン記録あり」フィルタ。

### 2-2. 編成詳細が貧弱
- 現状: コードと遭遇回数のみ。
- 鉄ヲタが見たい: **最終遭遇日、初遭遇日、遭遇場所マップ、添付写真ギャラリー、最後のラストラン記録**。
- 「自分とこの編成の物語」が見えない。

### 2-3. タイムラインのグルーピング硬直
- 現状: 年別固定。
- 必要: 月別 / 形式別 / 路線別 の切替。

### 2-4. 駅名・路線名がフリーテキスト
- 「新宿駅」「JR新宿」「新宿」など表記ゆれが発生し、後で集計できなくなる。
- 課題: 駅マスタの整備（v2.0候補）。今は最低限「最近入力した候補」のサジェストでもマシになる。

### 2-5. 統計画面の不在
- 「累計乗車距離」「最頻出形式」「最頻出路線」「最も足を運んだ駅」「年間遭遇数推移」は鉄ヲタの自己満足の中核。
- 現状: ウィジェットに「今月の遭遇数」しかない。

### 2-6. 乗車セッションがアプリ強制終了で消える
- `RideManager` は @Observable の State のみ。アプリ落としたら状態消失。
- 修正: `UserDefaults` に永続化、起動時復元。

### 2-7. iCloud同期状態が不可視
- 「ちゃんと同期されてる？」がガチ鉄の不安。設定画面に **最終同期時刻** が必要。

### 2-8. データエクスポートの不在
- 「自分のデータをロックインされる」のはガチ鉄が一番嫌う。**JSON書き出し**は信頼の証明。

### 2-9. 写真EXIFから位置情報を取らない
- 撮影後の追加記録時、写真の位置を手入力させるのは怠慢。EXIFから自動取得すべし。

### 2-10. 保存時のフィードバックが地味
- 単に閉じるだけ。**廃車進行中編成を記録した時の特別ハプティクス**など、情緒設計の余地。

### 2-11. 接近アラートで「編成」を指定できない
- 現状ウォッチリストUIは形式名のみ。`WatchItem.targetFormationCode` はモデルにあるのに UI で使われていない。「N102編成だけ追いたい」が叶わない。

### 2-12. オンボーディングなし
- 初回起動でいきなり空っぽの図鑑。寂しさで離脱する。最初の3タップ体験を用意すべき。

---

## 3. ガチ鉄向け「かゆいところ」

- **廃車32日前にあの編成を撮った** という情緒バッジ
- **コレクション率100%** の形式に「コンプ」金バッジ
- **ラストラン記録**を年表で永久ハイライト
- 編成カードの背景に **路線のラインカラー** を反映（山手線=緑、京急=赤、京阪=緑/オレンジなど）
- **「あなたがこの形式に初めて出会ったのは…」** という追想表示

---

## 4. 今すぐ手を付ける改修一覧

優先度高、無料で実装可。今回のコミットで対応する：

1. **モデル拡張** Sighting に `headmark`, `livery`, `weather`, `trainNumber`, `kind`, `carNumber` 追加（CloudKit互換にデフォルト値あり）
2. **シードデータ修正** 所属区所カナの整合（ミツ/トウ/マト/カワ/クラ/サイ等）
3. **検索・フィルタ** 図鑑に検索バー＋事業者フィルタ＋「未遭遇のみ」「廃車進行中のみ」
4. **編成詳細強化** 最終遭遇日・回数・関連Sighting一覧・コンプリート率達成バッジ
5. **タイムライングルーピング切替** 年/月/形式
6. **統計タブ追加** 累計距離・最頻出形式・最頻出路線・年別推移
7. **乗車セッション永続化** UserDefaultsに保存
8. **iCloud同期状態表示** 最終更新時刻と「100%ローカル」ラベル
9. **データエクスポート** JSON書き出し（設定画面）
10. **EXIF位置自動取得** AddSighting
11. **接近アラートUI拡充** 編成番号での追加
12. **ハプティクス** 記録保存時、廃車進行中記録時
13. **オンボーディング** 初回起動ウェルカム3画面

これらが入れば「ガチ鉄目線で見ても、無料アプリでよくここまで」のラインに到達する。
