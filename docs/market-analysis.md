# 鉄道アプリ市場分析サマリー（TetsuLog 企画の根拠）

最終更新: 2026-06-07

App Storeの鉄道関連アプリを5領域・約60サービスで調査した結論。TetsuLogのポジショニング根拠。

---

## 1. 市場構造：二極化

| 領域 | 支配者 | 理由 |

## 2. ユーザー不満の核心（既存レビュー横断）

| 不満 | 該当 | TetsuLogの解 |
|---|---|---|
| ガチャ重課金 | 駅メモ/鉄道パークZ | ガチャなし・買い切り |
| キャラ偏重で鉄道要素薄い | 駅メモ | 編成個体に振り切り |
| UI改悪・機能削除 | 駅探/Yahoo!/東武 | Apple純正UXで品質担保 |
| 公式アプリ断片化(1人5〜10個) | JR東6個/JR西4個/関東12社 | 横断はしないが個人記録で一元化 |
| 乗りつぶしオンラインのアプリ不在 | 開発者「個人では不可能」と公言 | ネイティブ実装で需要を直撃 |

---

## 3. TetsuLog のポジショニング

- **ターゲット**: 駅メモのキャラに違和感を持つガチ層 + 乗りつぶしオンライン現役層 + 撮り鉄。
- **コアバリュー**: 「キャラを買うんじゃなく、自分の鉄道人生を記録する」。
- **差別化**: 編成番号レベルの個体記録、撮影地×順光、廃線、ライブアクティビティ、接近アラート。
- **防御性**: ①ユーザー記録の蓄積、②Apple純正OS統合の体験品質、③プライバシー設計の信頼。
- **制約適合**: 完全無料運用（サーバーなし）・個人開発・プライバシー第一の3条件下で実装可能な機能のみ採用。

## 4. 10案からの取捨選択

| 案 | 採否 | 理由 |
|---|---|---|
| ④ 車両個体ロガー | ✅ コア | 無料・需要明確・防御性最強 |
| ⑧ 個人史タイムライン | ✅ コア | 同一データの別ビュー、追加コスト小 |
| ⑤ 撮影地×順光 | ✅ 採用 | 端末内計算で無料実装可 |
| ⑥ 廃線探索 | ✅ 採用 | 静的GeoJSON同梱で実装可 |
| ① AI識別 | ⚠️ OCRのみ | Live Textで編成番号読取。AI形式判定は不可 |
| ⑨ 多言語 | ⚠️ 手動翻訳 | String Catalogで対応、AI翻訳はしない |
| ③ 在線可視化 | ⚠️ 接近アラートに縮小 | GPS×静的ダイヤの端末内予測に限定 |
| ② 走行音SNS | ❌ | 権利・サーバーで無料運用不可 |
| ⑦ ライブカメラ | ❌ | 配信契約が個人不可 |
| ⑩ マッチング | ❌ | モデレーション負荷 |
