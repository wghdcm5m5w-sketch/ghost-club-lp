# TetsuLog v1.1 開発バックログ

v1.0 リリース成功後の開発サイクル。v1.0で先送りした項目・無効化した機能・第3次レビューで「将来課題」とした項目を構造化。

優先度: 🔴 高 / 🟡 中 / 🟢 低 / 🔵 メンテ

---

## v1.1 スコープ（提案）

短いサイクルで価値を出すため、v1.1は以下に絞る：
- **無効化した機能の復活（App Intents）** — v1.0で約束した機能を果たす
- **約束と実装の不整合解消（駅名正規化・写真バックアップ言及）** — 第3次レビュー残務
- **実機スクショ差し替え** — マーケ最強化

v2.0以降の大きな拡張（iPadレイアウト・Watch機能拡張・録音タグ等）は別サイクルへ。

---

## 🔴 高優先（v1.1 で対応）

### #1 App Intents 復活
- `app/TetsuLog/Features/TetsuLogIntents.swift` を `29e320c` の状態に復元
- 構造はそのまま：`LogSightingIntent` / `StartRideIntent` / `TetsuLogShortcuts`
- `ENABLE_APP_SHORTCUTS_FLEXIBLE_MATCHING = NO` が project.pbxproj に既に入っているので、SSUTraining でコケない見込み
- 検証: Xcode で Archive まで通るか／Siriから「テツログで記録」と話して動くか

### #2 駅名・路線名の正規化
- 集計時に「JR新宿」「新宿駅」「新宿」を同一視
- ヘルパー: `String.normalizedStationName` を追加
  - 末尾の「駅」を除去
  - 先頭の事業者プレフィックス（JR/小田急/京急 etc）を除去
- 適用先: `Statistics.topStations`、SuggestField のマッチング
- 既存記録は壊さない（保存時は原文のまま、集計時のみ正規化）

### #3 実機スクショ差し替え（手動）
- 6.9"（iPhone 17 Pro Max・1320×2868）と 6.5"（1242×2688）の両方
- シミュレータで ⌘S → 7枚を ASC アップロード
- マーケ用キャプション加工版（現在のもの）は副案として残す

## 🟡 中優先（v1.1 余力があれば）

### #4 写真バックアップの誠実化
- 設定の「JSONで書き出す」隣に **「写真をまとめて書き出す」** ボタンを追加
- アクション: `Documents/photos/` を ZIP 化 or 連続シェアで端末に保存
- ZIP は Foundation のみだと扱いづらい → **共有シートに複数ファイル渡す** が一番素直

### #5 重複記録のソフト警告
- AddSightingView で「同じ日・同じ編成の記録が直近にあります」を表示
- 拒否はしない（意図的な複数記録もあるため）
- 設定で「重複を警告する」トグル

### #6 CLMonitor 撮影地ジオフェンス・現在地ベース20件
- `ApproachMonitor` で全撮影地のうち**現在地から近い順に20件**だけ監視登録
- 位置が大きく動いたら再選定
- 既存ロジックを `selectNearestSpots(_:limit:)` で挟む
